### 3. Vercel（部署前端）
- 访问：https://vercel.com
- 点击：**Continue with GitHub**
- ✅ 一键登录，不用填任何信息！

点击绿色按钮：**Create repository**

### 2. 运行上传脚本

**在终端运行：**
```bash
cd /home/jack/ZRJ

```

按提示操作即可！

## 🌐 第四步：部署前端到 Vercel（10分钟）

### 1. 新建项目

**访问：** https://vercel.com/new

### 2. 导入仓库

找到并点击：`smart-teleprompter`

### 3. 配置项目

填写：
```
Framework Preset: Create React App
Root Directory: frontend
Build Command: npm run build
Output Directory: build
```

### 4. 添加环境变量

点击 **Environment Variables**

添加：
```
Name: REACT_APP_BACKEND_URL
Value: [刚才复制的 Railway 地址]
```

比如：`https://xxx.up.railway.app`

### 5. 点击部署

点击蓝色按钮：**Deploy**

⏰ **等待 3-5 分钟...**

### 6. 完成！

看到 🎉 "Congratulations" 就成功了！

点击 **Visit** 或复制域名

## ❓ 常见问题

### Q1: 上传代码时要求输入密码

**A:** 不是你的 GitHub 登录密码！需要 Personal Access Token：

1. 访问：https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. Note: 填 `smart-teleprompter`
4. Expiration: 选 `90 days`
5. 勾选：✅ **repo**（最上面那个）
6. 点击 **Generate token**
7. **复制生成的 token**（只显示一次！）
8. 回到终端，密码处粘贴这个 token

### Q2: Railway 部署失败

**检查：**
1. 环境变量是否正确设置？
2. Root Directory 是否设置为 `backend`？
3. 查看 **Deploy Logs** 看具体错误

### Q3: Vercel 部署失败

**检查：**
1. Root Directory 是否设置为 `frontend`？
2. 环境变量 `REACT_APP_BACKEND_URL` 是否正确？
3. 后端地址是否以 `https://` 开头？

### Q4: 手机上无法使用麦克风

**原因：** 浏览器需要 HTTPS 才能访问麦克风

**解决：**
- ✅ Vercel 自动提供 HTTPS
- 确保用 `https://` 开头的地址访问

### Q5: 能分享给朋友用吗？

**当然可以！** 部署后：

- ✅ 不需要你的电脑开机
